Twitter Bootstrap for PrototypeJS

Overview

Twitter Bootstrap is collection of HTML, CSS, and Javascript files that allow a developer to easily implement a clean looking design using elements that will look consistent across all browsers. However Twitter Bootstrap's Javascript components require jQuery and for some developers adding another Javascript library is not an option. This fork of Twitter Bootstrap implements all* of the Javascript components in PrototypeJS and will polyfill Script.aculo.us Effects support for browsers that do not support CSS3 Transitions (Internet Explorer 9 and below).
Affix, ScrollSpy, and Typeahead Javascript components have not been ported as of yet.

Usage

Include the CSS file and the Javascript file in your <head> 
<link rel="stylesheet" type="text/css" href="bootstrap.min.css" />
<script type="text/javascript" src="bootstrap_prototype.min.js"></script>

BootStrap.Modal

Define your modal div like this 
<div class="modal hide fade">
 <div class="modal-header">
  <button type="button" class="close" data-dismiss="modal" aria-hidden="true">×</button>
   <h3>BootStrap.Modal Header</h3>
  </div>
  <div class="modal-body">
   <p>Duis mollis, est non commodo luctus, nisi erat porttitor ligula, eget lacinia odio sem nec elit. Aenean eu leo quam. Pellentesque ornare sem lacinia quam venenatis vestibulum.</p>
  </div>
  <div class="modal-footer">
   <a href="#" class="btn" data-dismiss="modal">OK that looks nice</a>
 </div>
</div>
Using the built-in observers the script will automatically look for elements with the 'data-toggle' attribute and create a new BootStrap.Modal() object out of the target 
<a href="#mymodal" role="button" class="btn" data-toggle="modal">Launch modal</a>
Launch modal with built-in observers

Or you can setup the modal manually like this 
var mymodal_var = new BootStrap.Modal('mymodal');
and then trigger it with Javascript 
mymodal_var.show()
//or if you don't have the original object
$('mymodal').retrieve('bootstrap:modal').show()
Launch modal manually with Javascript

Options for BootStrap.Modal()

Name type default description
backdrop boolean true Includes a modal-backdrop element. Alternatively, specify static for a backdrop which doesn't close the modal on click.
keyboard boolean true Closes the modal when escape key is pressed
show boolean true Shows the modal when initialized.
remote path false

If a remote url is provided, content will be loaded via Ajax.Updater() method and injected into the .modal-body. If you're using the data api, you may alternatively use the href tag to specify the remote source. An example of this is shown below:

	<a data-toggle="modal" href="remote.html" data-target="#modal">click me</a>

Methods of the BootStrap.Modal() object

object.show()
Triggers the modal div to be shown and any transistions to happen. You can pass the options object to .show() as well.

object.hide()
Triggers the modal div to be hidden and any transistions to happen.

object.toggle()
Will show() if hidden, will hide() if shown.

BootStrap.Dropdown

The BootStrap.Dropdown object is usually never needed to be called manually as the built-in observers handle the setup. Only if you do not set the attributes would you ever need to manually trigger the dropdown menus.

Here is a simple example with 1 drop down and 1 menu option 
<div class="navbar navbar-static">
 <div class="navbar-inner">
  <div class="container" style="width: auto;">
   <ul class="nav">
    <li><a href="#" rolw="button" class="dropdown-toggle" data-toggle="dropdown">Drop Me <b class="caret"></b></a>
     <ul class="dropdown-menu">
      <li><a href="#">Pick Me</a></li>
     </ul>
    </li>
   </ul>
  </div>
 </div>
</div>
